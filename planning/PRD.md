# 克隆地球官网 PRD v1

## 1. 项目定位

这是“克隆地球（Earth Clone）”项目的官方网站，目标是承载三类内容：

1. **官网展示**
   - 项目愿景
   - 技术路线
   - 核心模块
   - Roadmap
   - 品牌视觉
2. **技术博客**
   - 开发日志
   - 技术分析
   - 调研笔记
   - 架构思考
3. **知识库**
   - 稳定沉淀的技术文档
   - 地形 / 建筑 / 道路 / 流式加载 / 数据处理等专题

---

## 2. 产品目标

网站第一阶段的目标不是做 CMS，而是快速上线一个成熟、可持续维护的项目站点。

重点：
- 让访客第一眼知道这是“克隆地球”项目
- 能持续输出博客与知识库内容
- 具备良好的技术项目气质和 SEO 能力
- 为未来多语言和视觉升级留空间

---

## 3. 技术方案

- **站点框架**：Docusaurus
- **语言**：TypeScript
- **部署**：Vercel
- **评论系统**：Giscus
- **内容管理**：Markdown / MDX + Git
- **默认语言**：中文
- **未来预留**：Docusaurus i18n

第一阶段**不做**：
- 自定义 CMS
- 自定义编辑器
- 自建评论系统
- 自建用户系统
- 复杂后端
- 重型 3D 交互首页

---

## 4. 页面结构

### 4.1 首页 `/`

首页必须弱化模板感，强调品牌感与项目感。

#### 首页包含区块
1. **Hero 区**
   - 标题：克隆地球
   - 一句话副标题
   - CTA：了解项目 / 阅读博客 / 查看知识库
2. **主视觉区**
   - 使用已生成的地球克隆宣传海报图
3. **项目简介区**
   - 项目要解决什么问题
   - 为什么值得做
   - 技术路线概览
4. **核心模块区**
   - 地形
   - 建筑
   - 道路
   - 水域 / 植被 / 动态交通（规划中）
5. **最新内容区**
   - 最近 3 篇博客
   - 最近 3 篇知识库文章
6. **Roadmap 区**
   - 当前阶段
   - 下一步计划

### 4.2 博客 `/blog`

用途：
- 开发日志
- 技术文章
- 调研总结
- 设计决策

要求：
- 标签
- 作者信息
- 摘要
- 发布时间
- Giscus 评论
- SEO 友好

### 4.3 知识库 `/docs`

知识库与博客分开。

初始知识库结构：
- 项目总览
- 架构设计
- 地形系统
- 建筑系统
- 道路系统
- 流式加载
- 数据流程
- 参考资料

### 4.4 About `/about`

内容包括：
- 项目背景
- 技术方向
- 当前阶段
- 联系方式 / GitHub

---

## 5. 评论系统

采用 **Giscus**。

原因：
- 开源
- 基于 GitHub Discussions
- 无需自建评论账户系统
- 适合技术社区
- 集成成本低

---

## 6. 多语言策略

第一阶段：
- 只做中文
- 目录和配置按 Docusaurus i18n 方式组织

第二阶段：
- 增加英文内容
- 使用 Docusaurus 原生 i18n 能力扩展

---

## 7. 视觉风格

关键词：
- 暗色
- 科技感
- 数字地球
- 工程项目气质
- 稳重，不浮夸

视觉元素建议：
- 地球 / 经纬网 / 线框 / 轨道线
- 深蓝 / 青色 / 橙色点缀
- 首页重点展示宣传海报
